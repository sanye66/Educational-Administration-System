# 教育管理系统 - 系统架构概览

## 1. 系统简介

教学管理系统（Educational Administration System）是一个基于 Spring Cloud Alibaba 微服务架构的综合性教育管理平台。系统涵盖学生管理、教师管理、课程管理、成绩管理、排课管理、考试管理、选课管理、教学评价、毕业设计等核心业务功能。

## 4. 核心业务模块

### 4.1 用户与权限模块

- **用户服务 (edu-user)**：系统用户管理，包括管理员、教师、学生账号
- **认证服务 (edu-auth)**：JWT 令牌发放与验证，权限控制

### 4.2 教学资源模块

- **教师服务 (edu-teacher)**：教师信息管理、职称评定
- **学生服务 (edu-student)**：学生信息管理、学籍管理
- **课程服务 (edu-course)**：课程信息、学分管理
- **教室服务 (edu-classroom)**：教室资源管理、预约

### 4.3 教学运行模块

- **选课服务 (edu-selection)**：学生选课、退课管理
- **排课服务 (edu-schedule)**：课程安排、教室分配
- **考试服务 (edu-exam)**：考试计划、考场安排
- **成绩服务 (edu-grade)**：成绩录入、绩点计算

### 4.4 教学评价模块

- **评价服务 (edu-evaluation)**：学生评教、教学质量分析

### 4.5 毕业管理模块

- **毕业设计服务 (edu-graduation)**：毕设题目、选题管理、答辩安排

### 4.6 AI 能力模块

- **AI 网关服务 (edu-ai-gateway)**：AI 能力统一接入、智能问答

## 5. 数据流说明

### 5.1 用户访问流程

```
用户登录 → edu-auth (颁发 Token) → 前端保存 Token
    ↓
请求 API → edu-gateway (路由鉴权) → 业务服务处理
    ↓
返回结果 → edu-gateway → 前端
```

### 5.2 业务数据流程

```
前端请求 → Gateway → 业务服务 → 数据库
                    ↓
               Redis 缓存
                    ↓
               其他服务 (Feign)
```

## 6. 部署架构

### 6.1 开发环境

- 所有服务运行在单机或 Docker Compose 环境
- 使用 H2 内存数据库或本地 MySQL
- Nacos 单节点

### 6.2 生产环境

- Kubernetes 集群部署
- MySQL 主从复制
- Redis 集群
- Nacos 集群
- 使用 CI/CD 自动化部署

## 7. 监控体系

- **Spring Boot Admin**：服务状态监控
- **Prometheus + Grafana**：指标收集与可视化
- **SkyWalking / Zipkin**：链路追踪
- **ELK**：日志收集与分析
