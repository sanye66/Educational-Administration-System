# 命名规范

本文档定义了教学管理系统的命名规范，适用于代码开发、数据库设计、API 命名等场景。

---

## 1. Java 命名规范

### 1.1 包名 (Package)

- 全部小写
- 禁止使用中文
- 多层级使用 `.` 分隔
- 格式：`com.{公司}.{项目}.{模块}`

**示例：**
```java
package com.edu.user.controller;
package com.edu.course.service.impl;
```

### 1.2 类名 (Class)

- 大驼峰命名 (PascalCase)
- 使用名词
- 禁止使用拼音缩写

**示例：**
```java
public class UserController { }
public class CourseServiceImpl { }
public class StudentMapper { }
```

### 1.3 方法名 (Method)

- 小驼峰命名 (camelCase)
- 动词或动词短语开头

**示例：**
```java
public User getUserById(Long id) { }
public List<Course> listCoursesByTeacherId(Long teacherId) { }
public void saveStudent(Student student) { }
```

### 1.4 变量名 (Variable)

- 小驼峰命名
- 禁止使用单字符（循环变量除外）
- 布尔类型加前缀 `is`、`has`、`can`

**示例：**
```java
private Long userId;
private String studentName;
private boolean isActive;
private boolean hasPermission;
```

### 1.5 常量 (Constant)

- 全大写
- 使用下划线分隔
- 添加 `LOGGER` 用于日志

**示例：**
```java
private static final Logger LOGGER = LoggerFactory.getLogger(UserController.class);
private static final int MAX_PAGE_SIZE = 100;
private static final String DEFAULT_CHARSET = "UTF-8";
```

### 1.6 接口命名

- 使用 `I` 前缀
- 实现类使用 `Impl` 后缀

**示例：**
```java
public interface IUserService { }
public class UserServiceImpl implements IUserService { }
```

## 2. 数据库命名规范

### 2.1 表名

- 小写字母 + 下划线
- 前缀与业务模块对应
- 使用单数名词

**示例：**
```sql
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY
);

CREATE TABLE course_info (
    id BIGINT PRIMARY KEY
);
```

### 2.2 字段名

- 小写字母 + 下划线
- 禁止使用关键字
- 主键统一使用 `id`

**示例：**
```sql
user_name VARCHAR(50)
create_time DATETIME
update_time DATETIME
is_deleted TINYINT
```

### 2.3 索引命名

- 主键：`pk_{表名}`
- 唯一索引：`uk_{表名}_{字段名}`
- 普通索引：`idx_{表名}_{字段名}`

**示例：**
```sql
UNIQUE KEY uk_user_username (username)
KEY idx_student_id (student_id)
```

## 3. API 命名规范

### 3.1 URL 路径

- 小写字母
- 使用 `-` 分隔
- 名词复数形式

**示例：**
```
GET    /api/users
GET    /api/courses/{id}
POST   /api/students
PUT    /api/teachers/{id}
DELETE /api/courses/{id}
```

### 3.2 方法命名

| 操作 | 方法 | 说明 |
|------|------|------|
| 查询 | get / list | get 获取单个，list 获取列表 |
| 新增 | save / create | save 通用，create 强调创建 |
| 更新 | update | 更新资源 |
| 删除 | delete | 删除资源 |

## 4. 前端命名规范

### 4.1 组件命名

- 大驼峰 (PascalCase)
- 文件名与组件名一致
- 页面组件以 `Page` 结尾

**示例：**
```typescript
// UserManage.tsx
export const UserManage: React.FC = () => { }

// StudentListPage.tsx
export const StudentListPage: React.FC = () => { }
```

### 4.2 样式命名

- 小写字母 + 中划线 (kebab-case)
- BEM 命名风格

**示例：**
```css
.user-list__item {}
.user-list__item--active {}
```

### 4.3 目录结构

```
src/
├── components/       # 通用组件
├── pages/           # 页面组件
├── api/             # API 接口
├── utils/           # 工具函数
└── styles/          # 样式文件
```

## 5. Git 命名规范

### 5.1 分支命名

- 功能分支：`feature/{功能名}`
- 修复分支：`fix/{问题描述}`
- 发布分支：`release/{版本号}`
- 热修复分支：`hotfix/{问题描述}`

**示例：**
```
feature/user-login
fix/database-connection
release/1.0.0
```

### 5.2 提交信息

- 格式：`[类型] 描述`
- 类型：feat、fix、docs、style、refactor、test、chore

**示例：**
```
[feat] 添加用户登录功能
[fix] 修复成绩查询空指针异常
[docs] 更新 API 文档
```
