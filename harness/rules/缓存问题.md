# 缓存问题

本文档定义了教学管理系统中 Redis 缓存的使用规范与常见问题解决方案。

---

## 1. 缓存设计原则

### 1.1 缓存使用场景

| 场景 | 适用 | 不适用 |
|------|------|--------|
| 数据变化频率 | 低 | 高 |
| 数据访问频率 | 高 | 低 |
| 数据一致性要求 | 低 | 高 |
| 数据量 | 小 | 大 |

### 1.2 缓存策略

```java
@Service
public class CacheStrategy {

    /**
     * 缓存穿透：布隆过滤器 + 空值缓存
     */
    public User getUserById(Long id) {
        String cacheKey = "user:" + id;
        
        // 1. 查询布隆过滤器
        if (!bloomFilter.mightContain(id)) {
            return null;
        }
        
        // 2. 查询缓存
        User user = redisTemplate.opsForValue().get(cacheKey);
        if (user != null) {
            return user;
        }
        
        // 3. 查询数据库
        user = userMapper.selectById(id);
        if (user != null) {
            redisTemplate.opsForValue().set(cacheKey, user, 30, TimeUnit.MINUTES);
        } else {
            // 空值缓存，防止穿透
            redisTemplate.opsForValue().set(cacheKey, "", 5, TimeUnit.MINUTES);
        }
        
        return user;
    }
}
```

## 2. 缓存一致性问题

### 2.1 缓存双写

数据更新时，先更新数据库，再删除缓存：

```java
@Service
public class UserServiceImpl implements IUserService {

    private final RedisTemplate<String, Object> redisTemplate;

    @Transactional(rollbackFor = Exception.class)
    @Override
    public void updateUser(User user) {
        // 1. 先更新数据库
        userMapper.updateById(user);
        
        // 2. 删除缓存（而非更新）
        redisTemplate.delete("user:" + user.getId());
    }
}
```

### 2.2 延迟双删

解决并发更新导致的缓存不一致：

```java
@Transactional(rollbackFor = Exception.class)
@Override
public void updateUser(User user) {
    // 1. 先删除缓存
    redisTemplate.delete("user:" + user.getId());
    
    // 2. 更新数据库
    userMapper.updateById(user);
    
    // 3. 延迟删除缓存（500ms）
    redisTemplate.execute(
        (RedisCallback<?>) connection -> {
            Thread.sleep(500);
            redisTemplate.delete("user:" + user.getId());
            return null;
        }
    );
}
```

## 3. 缓存雪崩

### 3.1 问题描述

大量缓存同时失效，导致大量请求直接访问数据库。

### 3.2 解决方案

#### 方案一：随机过期时间

```java
public void setCache(String key, Object value) {
    // 基础过期时间 30 分钟 + 随机偏移量 0-5 分钟
    int expireTime = 1800 + new Random().nextInt(300);
    redisTemplate.opsForValue().set(key, value, expireTime, TimeUnit.SECONDS);
}
```

#### 方案二：逻辑过期

```java
@Data
public class CacheData<T> {
    private T data;
    private long expireTime;
    
    public boolean isExpired() {
        return System.currentTimeMillis() > expireTime;
    }
}
```

#### 方案三：永不过期 + 后台刷新

```java

    // 后台线程定时刷新缓存
```

## 4. 缓存击穿

### 4.1 问题描述

热点 key 失效瞬间，大量并发请求直接访问数据库。

### 4.2 解决方案

#### 方案一：互斥锁

```java
public User getUserById(Long id) {
    String cacheKey = "user:" + id;
    String lockKey = "lock:user:" + id;
    
    // 1. 查询缓存
    User user = redisTemplate.opsForValue().get(cacheKey);
    if (user != null) {
        return user;
    }
    
    // 2. 获取分布式锁
    Boolean acquired = redisTemplate.opsForValue()
        .setIfAbsent(lockKey, "1", 10, TimeUnit.SECONDS);
    
    if (Boolean.TRUE.equals(acquired)) {
        try {
            // 3. 双重检查
            user = redisTemplate.opsForValue().get(cacheKey);
            if (user != null) {
                return user;
            }
            
            // 4. 查询数据库
            user = userMapper.selectById(id);
            
            // 5. 写入缓存
            redisTemplate.opsForValue().set(cacheKey, user, 30, TimeUnit.MINUTES);
            
            return user;
        } finally {
            redisTemplate.delete(lockKey);
        }
    } else {
        // 等待后重试
        Thread.sleep(50);
        return getUserById(id);
    }
}
```

#### 方案二：逻辑过期

```java
public User getUserById(Long id) {
    String cacheKey = "user:" + id;
    CacheData<User> cacheData = redisTemplate.opsForValue().get(cacheKey);
    
    if (cacheData == null) {
        // 缓存为空，加载数据到缓存（不设置过期时间）
        User user = userMapper.selectById(id);
        CacheData<User> newCacheData = new CacheData<>();
        newCacheData.setData(user);
        newCacheData.setExpireTime(System.currentTimeMillis() + Long.MAX_VALUE);
        redisTemplate.opsForValue().set(cacheKey, newCacheData);
        return user;
    }
    
    // 命中，检查是否过期
    if (!cacheData.isExpired()) {
        return cacheData.getData();
    }
    
    // 已过期，尝试获取锁刷新
    String lockKey = "refresh:user:" + id;
    Boolean acquired = redisTemplate.opsForValue().setIfAbsent(lockKey, "1", 10, TimeUnit.SECONDS);
    if (Boolean.TRUE.equals(acquired)) {
        // 异步刷新缓存
        CompletableFuture.runAsync(() -> {
            User user = userMapper.selectById(id);
            cacheData.setData(user);
            cacheData.setExpireTime(System.currentTimeMillis() + 30 * 60 * 1000);
            redisTemplate.opsForValue().set(cacheKey, cacheData);
            redisTemplate.delete(lockKey);
        });
    }
    
    // 返回旧数据
    return cacheData.getData();
}
```

## 6. 大 Key 问题

### 6.1 问题识别

```java
@Component
public class BigKeyScanner {

    @Autowired
    private RedisConnectionFactory connectionFactory;

    public void scanBigKeys() {
        Set<String> keys = redisTemplate.keys("*");
        for (String key : keys) {
            Long size = getKeySize(key);
            if (size > 1024 * 1024) { // 1MB
                log.warn("发现大 Key: {}, 大小: {} bytes", key, size);
            }
        }
    }
}
```

### 6.2 解决方案

#### 方案一：拆分大 Key

```java
// ❌ 大 List
redisTemplate.opsForList().rightPushAll("user:1:friends", friends);

// ✅ 拆分
for (int i = 0; i < friends.size(); i++) {
    redisTemplate.opsForList().rightPush("user:1:friends:" + (i / 100), friends.get(i));
}
```

#### 方案二：压缩

```java
// 序列化时启用压缩
public <T> void setCache(String key, T value) {
    byte[] serialized = serialize(value);
    byte[] compressed = compress(serialized);
    redisTemplate.opsForValue().set(key, compressed);
}
```

## 7. 缓存使用规范

### 7.1 Key 命名规范

```
格式：{项目}:{模块}:{业务}:{标识}

示例：
edu:user:info:1001          # 用户信息
edu:course:list:teacher:5   # 教师的课程列表
edu:student:cache:grade:10   # 学生成绩缓存
```

### 7.2 过期时间规范

| 数据类型 | 过期时间 |
|----------|----------|
| 用户信息 | 30 分钟 |
| 列表数据 | 10 分钟 |
| 配置数据 | 1 小时 |
| 验证码 | 5 分钟 |
| Token | 2 小时 |

### 7.3 序列化配置

```java
@Configuration
public class RedisConfig {

    @Bean
    public RedisTemplate<String, Object> redisTemplate(
            RedisConnectionFactory connectionFactory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(connectionFactory);

        // JSON 序列化
        Jackson2JsonRedisSerializer<Object> serializer = 
            new Jackson2JsonRedisSerializer<>(Object.class);
        
        StringRedisSerializer stringSerializer = new StringRedisSerializer();

        template.setKeySerializer(stringSerializer);
        template.setValueSerializer(serializer);
        template.setHashKeySerializer(stringSerializer);
        template.setHashValueSerializer(serializer);

        template.afterPropertiesSet();
        return template;
    }
}
```
