# 公司标准

本文档定义了教学管理系统开发、部署、运维等环节的公司标准规范。

---

## 1. 代码规范

### 1.1 开发规范

| 项目 | 标准 |
|------|------|
| 开发工具 | IDEA / VSCode |
| 代码格式 | Alibaba Java Coding Guidelines |
| 代码审查 | PR 必须经过 1 人以上 review |
| 提交规范 | 提交信息必须符合 Conventional Commits |
| 测试要求 | 核心业务必须有单元测试 |

### 1.2 代码审查标准

```markdown
## PR 审查清单

- [ ] 代码符合命名规范
- [ ] 无硬编码密码/密钥
- [ ] 无明显的性能问题
- [ ] 有必要的注释
- [ ] 有对应的单元测试
- [ ] 已更新相关文档
```

### 1.3 代码覆盖率要求

| 模块 | 最低覆盖率 |
|------|------------|
| 核心业务 | 70% |
| Controller | 50% |
| Service | 70% |
| Mapper | 80% |

## 3. 安全标准

### 3.1 敏感信息管理

```yaml
# ✅ 正确：使用环境变量或密钥管理服务
spring:
  datasource:
    password: ${DB_PASSWORD}

# ❌ 错误：硬编码密码
spring:
  datasource:
    password: my_secret_password
```

### 3.2 接口安全要求

| 要求 | 说明 |
|------|------|
| 身份认证 | 所有接口需通过身份验证 |
| 参数校验 | 必须对输入参数进行校验 |
| SQL 注入 | 必须使用预编译语句 |
| XSS 攻击 | 必须对输出进行转义 |
| CSRF 防护 | 必须启用 CSRF 防护 |

### 3.3 日志脱敏规范

```java
@Service
public class UserServiceImpl {

    public void saveUser(User user) {
        // ✅ 正确：脱敏处理
        LOGGER.info("保存用户, username: {}, phone: {}", 
                   DesensitizedUtil.desensitize(user.getUsername()),
                   DesensitizedUtil.mobilePhone(user.getPhone()));
        
        // ❌ 错误：记录敏感信息
        LOGGER.info("保存用户, password: {}", user.getPassword());
    }
}
```

## 4. 部署标准

### 4.1 环境划分

| 环境 | 用途 | 数据库 | 访问要求 |
|------|------|--------|----------|
| DEV | 开发测试 | 本地/开发库 | 内部网络 |
| TEST | 集成测试 | 测试库 | 内部网络 |
| PRE | 预发布 | 预发布库 | 内部网络 + 部分外部 |
| PROD | 生产 | 生产库 | 仅生产网络 |

### 4.2 镜像构建标准

```dockerfile
# 基础镜像
FROM maven:3.9-eclipse-temurin-11 AS builder

# 复制源码
COPY pom.xml .
COPY src ./src

# 构建
RUN mvn clean package -DskipTests

# 运行镜像
FROM eclipse-temurin:11-jre-alpine
WORKDIR /app
COPY --from=builder /target/*.jar app.jar

# 非 root 用户运行
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser

ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 4.3 部署配置标准

```yaml
# 生产环境资源配置
resources:
  requests:
    memory: "512Mi"
    cpu: "250m"
  limits:
    memory: "1Gi"
    cpu: "1000m"

# 健康检查
livenessProbe:
  httpGet:
    path: /actuator/health/liveness
    port: 8080
  initialDelaySeconds: 60
  periodSeconds: 10
  failureThreshold: 3

readinessProbe:
  httpGet:
    path: /actuator/health/readiness
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 5
  failureThreshold: 3
```

## 5. 监控标准

### 5.1 监控指标

| 指标类型 | 监控项 | 告警阈值 |
|----------|--------|----------|
| 基础指标 | CPU 使用率 | > 80% |
| 基础指标 | 内存使用率 | > 85% |
| 基础指标 | 磁盘使用率 | > 90% |
| 应用指标 | QPS | < 10 或 > 1000 |
| 应用指标 | 响应时间 P99 | > 1000ms |
| 应用指标 | 错误率 | > 1% |
| 业务指标 | 登录失败次数 | > 10次/分钟 |

### 5.2 日志规范

```xml
<!-- logback 配置 -->
<configuration>
    <!-- 日志格式 -->
    <property name="LOG_PATTERN" 
              value="%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{50} - %msg%n"/>
    
    <!-- 按日期和级别分离 -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/edu-system.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>logs/edu-system.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
            <maxHistory>30</maxHistory>
            <timeBasedFileNamingAndTriggeringPolicy 
                class="ch.qos.logback.core.rolling.SizeAndTimeBasedFNATP">
                <maxFileSize>100MB</maxFileSize>
            </timeBasedFileNamingAndTriggeringPolicy>
        </rollingPolicy>
    </appender>
</configuration>
```

## 7. 流程标准

### 7.1 发布流程

```
需求评审
    ↓
技术设计
    ↓
代码开发
    ↓
代码审查 (PR)
    ↓
单元测试
    ↓
集成测试
    ↓
预发布验证
    ↓
生产发布
    ↓
发布确认
```

### 7.2 故障处理流程

```
发现故障
    ↓
故障定级 (P1-P4)
    ↓
故障响应
    ↓
故障排查
    ↓
故障修复
    ↓
故障恢复
    ↓
故障复盘
```

### 7.3 变更流程

```
变更申请
    ↓
技术评审
    ↓
变更审批
    ↓
变更实施
    ↓
变更验证
    ↓
变更关闭
```
